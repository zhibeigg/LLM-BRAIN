## Design Context

### Users
LLM-BRAIN is for developers, researchers, and power users operating a directed memory graph with multi-role LLM agents. They use it while asking tasks, learning new topics, debugging agent reasoning, and inspecting how memories influence model behavior.

### Brand Personality
Engineering-trustworthy, observable, brain-like. The interface should feel precise and reliable first, then surprising through clear real-time visualization rather than decorative spectacle.

### Aesthetic Direction
Dark, instrument-grade graph workspace with a balanced “neural field” overlay. During model communication, the right-side node graph should show information moving between User, Leader, Memory Graph, Agent, Tools, and Boss without hiding the underlying graph. Effects should read as telemetry and signal flow, not game-like decoration.

### Design Principles
- Make hidden agent coordination visible through event-driven signals.
- Preserve the memory graph as the primary artifact; enhancements are overlays, not distractions.
- Prefer lightweight SVG/CSS motion over continuous heavy rendering for daily reliability.
- Respect reduced-motion preferences with a clean static state.
- Use motion to clarify state transitions: routing, retrieval, generation, tool use, and validation.
